# Карта Геленджика (React + офлайн-кэш)

Одностраничное React-приложение с картой Геленджика (Leaflet + OpenStreetMap),
кастомными метками и левой панелью, которая открывается по клику на метку.
В панели можно добавлять фото из галереи iPhone — они сохраняются локально
в IndexedDB, поэтому доступны даже без интернета.

## Как это устроено

- **Один REST-эндпоинт**: `GET /api/markers` (FastAPI, папка `backend/`) отдаёт список меток.
- **Кэширование данных**: Service Worker (vite-plugin-pwa) кэширует тайлы карты (стратегия CacheFirst)
  и ответ `/api/markers` (стратегия NetworkFirst с фолбэком на кэш). Дополнительно сами метки
  дублируются в IndexedDB через `idb`, чтобы приложение работало даже при первом запуске без сети.
- **Фото**: хранятся как Blob прямо в IndexedDB (объектный стор `photos`), привязаны к `markerId`.
  Ничего не улетает на сервер — всё остаётся на устройстве.
- **Левая панель**: `SidePanel.tsx` — открывается при клике на маркер, показывает заголовок,
  описание, сетку фото и кнопку "Добавить файл из галереи" (`<input type="file" accept="image/*">`).

## Запуск

### Backend (эндпоинт меток)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### Frontend
```bash
npm install
npm run dev
```
Открой `http://localhost:5173` в Safari на iPhone (в одной Wi-Fi сети с компом) —
или задеплой на любой статический хостинг (Vercel, Netlify, GitHub Pages).

## Установка на iPhone как офлайн-приложение

1. Открой сайт в Safari.
2. Нажми "Поделиться" → "На экран Домой".
3. Приложение станет полноценной PWA — после первого открытия с интернетом
   все тайлы карты, список меток и сам интерфейс закэшируются, и дальше
   всё будет работать даже в самолёте.

## Важные нюансы для офлайн-режима

- Тайлы карты кэшируются только для тех областей, которые ты реально открыл/проскроллил
  во время использования с интернетом — заранее "проскролль" весь Геленджик один раз.
  Если нужно гарантированно закэшировать конкретный регион целиком — рекомендую скачать
  тайлы через `mbtiles` и раздавать их локально (более сложный вариант, спрошу отдельно если нужно).
- Фото не требуют интернета вообще — они физически лежат в браузере.
- Если хочешь синхронизировать фото между устройствами — понадобится расширить backend
  эндпоинтом `POST /api/markers/{id}/photos` с загрузкой на сервер (сейчас его нет по ТЗ:
  "один эндпоинт" — оставил только GET).
